Evoke Section

Evoke templates used:

EKS_CE_RESULT_STATUS_E

Evoke on CLINICAL_EVENT where :

E1EKS_CE_RESULT_STATUS_E
CVAD removed: is in any of the statuses listed: Modified, Modified, Auth (Verified)

Description : Result status
Keywords : CLINICALEVENT

Logic Section

Logic templates used:

EKS_ENCOUNTER_DETAIL_L, EKS_CE_RESULT_INCOMING_REQ_L, EKS_ORDERS_FIND_L

L1EKS_ENCOUNTER_DETAIL_L
the patient's Encounter Type is listed INPATIENT for the same encounter as L?

Description : Encounter attributes
Keywords : ENCOUNTER
AND
L2EKS_CE_RESULT_INCOMING_REQ_L
the triggering request contains a Modified, Auth (Verified), Modified result for CVAD removed: that is equal to Yes and OPT_VALUE2

Description : Incoming numeric result value, status
Keywords : CLINICALEVENT
AND
L3EKS_ORDERS_FIND_L
orders exist whose primary mnemonic is CVAD dressing change, CVAD Port needle change, CVAD Hep Lock - Red, CVAD Hep Lock - White, CVAD Hep Lock - Purple, CVAD Hep Lock - Blue, CVAD Hep Lock - Brown, CVAD Hep Lock - Portacath Left, CVAD Hep Lock - Portacath Right, CVAD Hep Lock - Other, CVAD NAD change - Red, CVAD NAD Change - White, CVAD NAD Change - Purple, CVAD NAD Change - Blue, CVAD NAD Change - Brown, CVAD NAD Change - Portacath Left, CVAD NAD Change - Portacath Right, CVAD NAD Change - Other, CVAD IV set change - Red, CVAD IV set change - White, CVAD IV set change - Purple, CVAD IV set change - Blue, CVAD IV set change - Brown, CVAD IV set change - Portacath Left, CVAD IV set change - Portacath Right, CVAD IV set change - Other with order status of Ordered on the same encounter as L2 whose date criteria is OPT_QUAL OPT_TIME_NUM OPT_TIME_UNIT from the anchor OPT_ANCHOR_DT_TM.

Description : Orderable, status, attributes
Keywords : ORDER
